# Phase 4: Remix Frontend (Cloudflare Pages)

## Objective

Build the Remix SSR frontend deployed to Cloudflare Pages. It handles authentication via Supabase cookies, communicates with the Hono API via type-safe RPC, and uses Tailwind CSS v4 with shadcn/ui components.

## Dependencies

- Phase 1 complete (monorepo infrastructure)
- Phase 3 complete (API exports `AppType` for Hono RPC)

### Utilities

#### `app/utils/supabase.server.ts`

```typescript
import { createServerClient } from "@supabase/ssr";

export function createSupabaseClient(request: Request, env: Env) {
  const cookies = Object.fromEntries(
    request.headers.get("Cookie")?.split("; ").map((c) => c.split("=")) ?? []
  );

  return createServerClient(env.SUPABASE_URL, env.SUPABASE_ANON_KEY, {
    cookies: {
      getAll() {
        return Object.entries(cookies).map(([name, value]) => ({ name, value }));
      },
      setAll(cookiesToSet) {
        // Cookies are set via response headers in the loader/action
      },
    },
  });
}
```

Cookie handling needs careful implementation — `setAll` must return Set-Cookie headers on the response. The actual implementation should follow the `@supabase/ssr` Remix guide.

#### `app/utils/api.server.ts`

```typescript
import { hc } from "hono/client";
import type { AppType } from "@todo-supabase/api";

export function createApiClient(apiUrl: string, accessToken: string) {
  return hc<AppType>(apiUrl, {
    headers: {
      Authorization: `Bearer ${accessToken}`,
    },
  });
}
```

- Type-safe RPC client — all endpoints are fully typed via `AppType`.

### Routes

#### `app/routes/_index.tsx`

Redirect logic:
- If user has session → redirect to `/todos`
- If no session → redirect to `/login`

#### `app/routes/login.tsx`

- Uses `Card`, `CardHeader`, `CardContent`, `Input`, `Label`, `Button`
- Action: `supabase.auth.signInWithPassword({ email, password })`
- On success: redirect to `/todos`
- Link to `/signup` for new users

#### `app/routes/signup.tsx`

- Same layout as login
- Action: `supabase.auth.signUp({ email, password })`
- On success: redirect to `/login` (or `/todos` if auto-confirmed)

#### `app/routes/logout.tsx`

- Action-only route (no loader/component)
- Action: `supabase.auth.signOut()`
- Redirect to `/login`

#### `app/routes/todos.tsx`

- **Loader**: fetch todos via Hono RPC (`api.todos.$get()`)
- **Action**: handle create/toggle/delete based on form `intent` field
  - `create`: `api.todos.$post({ json: { title } })`
  - `toggle`: `api.todos[":id"].$patch({ param: { id }, json: { completed } })`
  - `delete`: `api.todos[":id"].$delete({ param: { id } })`
- **Component**: Input for new todo, list of todos with Checkbox and delete Button
- Logout button in header

#### `app/routes/auth.callback.tsx`

- Exchanges auth code for session (for OAuth or email confirmation flows)
- Loader: extract `code` from URL, call `supabase.auth.exchangeCodeForSession(code)`
- Redirect to `/todos`

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Vite plugin order | cloudflareDevProxy must be first; remix must be last |
| `@supabase/ssr` over auth-helpers | auth-helpers is deprecated |
| Cookie-based auth in Remix | SSR-standard; works with Cloudflare Pages |
| Hono RPC (`hono/client`) | Compile-time type safety; no manual fetch/type definitions |
| `@todo-supabase/api` as devDep | Only the type is needed; no runtime code ships to the browser |
| shadcn/ui new-york style | Source-owned components; no runtime dependency |
| No shared UI package | YAGNI — components live in `apps/web` only |
